# Nano Banana + Milvus + Neo4j 电商 AI 生图流水线

> 基于向量检索、知识图谱和 AIGC 的电商宣传图自动生成系统

---

## 项目背景

### 痛点问题

中小电商企业在商品宣传图制作中面临以下挑战：

- **成本高**：专业拍摄 + 后期修图，单张成本可达几十至上百元
- **周期长**：从新品上架到宣传图完成，通常需要数天时间
- **质量不稳定**：依赖设计师个人能力，风格难以统一

### 解决方案

本项目通过 **向量检索 + 知识图谱 + 多跳推理 + 两阶段检索 + 风格分析 + 图像生成 + 质量评估** 技术实现：

```
新品平铺图 → 两阶段检索(多跳推理+向量精排) → 提取拍摄风格 → AI 生成宣传图 → AI 质量评分
```

**核心价值**：
- 无需本地 GPU，所有模型通过 API 调用
- 端到端自动化，单张图片生成成本约 $0.014 (FLUX.2 Klein)
- 基于历史爆款数据，保证生成风格符合品牌调性
- **多跳推理**：通过风格相似性自动扩展候选集（3跳）
- **两阶段检索**：Neo4j 图谱过滤 + Milvus 向量精排
- **智能降级**：Neo4j 失败时自动切换到 Milvus 全量检索
- **本地缓存**：相同爆款组合自动复用风格分析结果，节省 API 费用
- AI 自动质量评估，确保生成图片可用性

### 第一性原则

**服装必须来自用户上传的图片**：系统生成的宣传图中，模特身上穿的服装必须是用户上传的原款服装。参考爆款图只用于提取**拍摄风格**（场景、光线、构图、氛围），不提供服装款式参考。

## 技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **向量数据库** | Milvus 2.4+ | 混合向量检索（Dense + Sparse + Scalar） |
| **图数据库** | Neo4j 5.0+ | 知识图谱、多跳推理 |
| **Embedding 模型** | nvidia/llama-nemotron-embed-vl-1b-v2 | 图文向量化（2048维）|
| **LLM 模型** | qwen/qwen3-vl-235b-a22b-thinking | 视觉风格分析 + 质量评估 |
| **图像生成** | black-forest-labs/flux.2-klein-4b | 宣传图生成 |
| **API 聚合** | OpenRouter | 统一模型调用接口 |
| **文本处理** | scikit-learn (TF-IDF) | 稀疏向量生成 |
| **图像处理** | Pillow | 图片加载、格式转换 |
| **后端框架** | FastAPI | RESTful API 服务 |
| **前端框架** | Vue 3 + Vite | Web 界面 |

### 检索模式对比

| 模式 | 说明 | 适用场景 | 优势 |
|------|------|---------|------|
| **两阶段检索** | Neo4j多跳推理 → Milvus向量精排 | 生产环境（推荐） | 召回率高、计算量小、支持风格扩展 |
| **混合检索** | Milvus + Neo4j 并行 → RRF融合 | 对比测试 | 结果多样、双重验证 |
| **循环检索** | 多轮查询重写 + 质量评分 | 混合检索模式 | 自动优化、解决无结果问题 |

### 多跳推理示例

假设查询 `elegant` 风格的 `midi_dress`：

```
第1跳: 匹配 elegant 风格
  → 找到 50 个 elegant 风格的商品（含 midi_dress、maxi_dress 等）

第2跳: 风格相似扩展
  → elegant 相似风格：romantic、classic、minimalist
  → 额外找到 30 个相似风格商品

第3跳: 跨品类扩展
  → elegant 风格的 Top 爆款（不限品类）
  → 额外找到 20 个高质量商品

候选集总计：100 个商品ID → 传递给 Milvus 做向量精排
```

## 快速开始

### 环境要求

- Python 3.9+
- Node.js 16+
- Milvus 2.4+ (Docker 或本地安装)
- Neo4j 5.0+ (Docker 或本地安装)
- OpenRouter API Key

### 1. 启动 Milvus

```bash
docker run -d --name milvus-standalone \
  -p 19530:19530 \
  -v $(pwd)/backend/milvus_data:/var/lib/milvus \
  milvusdb/milvus:latest
```

### 2. 启动 Neo4j

```bash
docker run -d \
  --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password \
  -v $(pwd)/neo4j_data:/data \
  neo4j:latest
```

### 3. 配置环境变量

创建 `.env` 文件：

```bash
OPENROUTER_API_KEY=your-key-here
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=12345678
```

### 4. 启动后端

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate  # Windows
pip install -r requirements.txt
python api.py
```

后端运行在 http://localhost:8000

### 5. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端运行在 http://localhost:3000

## AI 质量评估

### 生成图片质量评估

启用质量评估后，AI 裁判会对生成的图片进行多维度评分：

| 评分维度 | 说明 | 满分 |
|---------|------|------|
| `clothing_accuracy` | 服装准确性：与原图匹配度 | 5 |
| `pose_naturalness` | 姿势自然度：模特姿势和合身度 | 5 |
| `scene_quality` | 场景质量：背景/场景专业性 | 5 |
| `lighting_quality` | 布光质量：光线质量 | 5 |
| `commercial_value` | 商业价值：是否适合电商使用 | 5 |
| `average` | 平均分 | 5 |

**质量判断**：
- 平均分 ≥ 3.5：质量合格
- 平均分 < 3.5：建议重新生成
- 服装准确性 < 4：建议重新生成（违反第一性原则）

## 常见问题

**Q: Neo4j 连接失败？**

A: 检查 Neo4j 是否启动：`docker ps | grep neo4j`。系统会自动降级到 Milvus 检索。

**Q: 检索结果为空？**

A: 检查数据库是否已初始化。多跳推理会自动扩展候选集，增加召回率。

**Q: 两阶段检索和混合检索有什么区别？**

A: 两阶段��索先通过 Neo4j 获取候选集，再用 Milvus 精排；混合检索是并行检索后融合。两阶段检索计算量更小。
