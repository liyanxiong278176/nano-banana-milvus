# Nano Banana + Milvus 电商 AI 生图流水线

> 基于 LangGraph 多Agent 工作流、向量检索和 AIGC 的电商宣传图自动生成系统

---

## 项目背景

### 痛点问题

中小电商企业在商品宣传图制作中面临以下挑战：

- **成本高**：专业拍摄 + 后期修图，单张成本可达几十至上百元
- **周期长**：从新品上架到宣传图完成，通常需要数天时间
- **质量不稳定**：依赖设计师个人能力，风格难以统一

### 解决方案

本项目通过 **LangGraph 多Agent 工作流 + 循环检索状态机 + 智能降级 + 风格分析 + 图像生成** 技术实现：

```
新品平铺图 → 多Agent工作流 → 向量检索 → 提取拍摄风格 → AI 生成宣传图
```

**核心价值**：
- 无需本地 GPU，所有模型通过 OpenRouter API 调用
- 端到端自动化，单张图片生成成本约 $0.014
- 基于 LangGraph 的多Agent 编排，流程清晰可扩展
- **循环检索状态机**：最多3轮查询重写，LLM质量评估驱动优化
- **智能降级机制**：无结果时自动放宽过滤条件
- **本地缓存**：风格分析结果自动缓存，支持自动清理

### 第一性原则

**服装必须来自用户上传的图片**：系统生成的宣传图中，模特身上穿的服装必须是用户上传的原款服装。参考爆款图只用于提取**拍摄风格**（场景、光线、构图、氛围），不提供服装款式参考。

## 技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **工作流编排** | LangGraph 1.0+ | 多Agent 状态机编排 |
| **向量数据库** | Milvus 2.4+ | 混合向量检索（Dense + Sparse + Scalar） |
| **Embedding 模型** | nvidia/llama-nemotron-embed-vl-1b-v2 | 图文向量化（2048维）|
| **LLM 模型** | qwen/qwen3-vl-8b-instruct | 视觉风格分析 + 质量评估 |
| **图像生成** | black-forest-labs/flux.2-klein-4b | 宣传图生成 |
| **API 聚合** | OpenRouter | 统一模型调用接口 |
| **文本处理** | scikit-learn (TF-IDF) | 稀疏向量生成 |
| **后端框架** | FastAPI | RESTful API 服务 |
| **前端框架** | Vue 3 + Vite | Web 界面 |

## 快速开始

### 环境要求

- Python 3.9+
- Node.js 16+
- Milvus 2.4+ (Docker)
- OpenRouter API Key

### 1. 启动 Milvus

```bash
docker run -d --name milvus-standalone \
  -p 19530:19530 \
  -v $(pwd)/backend/milvus_data:/var/lib/milvus \
  milvusdb/milvus:latest
```

### 2. 配置环境变量

创建 `backend/.env` 文件：

```bash
# 必填：OpenRouter API Key（从 https://openrouter.ai/keys 获取）
OPENROUTER_API_KEY=your-openrouter-api-key-here

# 可选：模型层级配置（standard | high_quality | economy）
# OVERRUN_MODEL_TIER=standard

# 可选：缓存配置
# ENABLE_CACHE=true

# 可选：网络代理
# HTTP_PROXY=http://127.0.0.1:7890
# HTTPS_PROXY=http://127.0.0.1:7890
```

### 3. 初始化数据库

```bash
cd backend
python -c "
from main import FashionImagePipeline
pipeline = FashionImagePipeline()
pipeline._init_database()
"
```

### 4. 启动后端

```bash
cd backend
pip install -r requirements.txt
python api.py
```

后端运行在 http://localhost:8000

### 5. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端运行在 http://localhost:3000

## API 使用

### 上传并生成

```bash
curl -X POST "http://localhost:8000/api/upload" \
  -F "file=@new_product.jpg" \
  -F "category=midi_dress" \
  -F "style=elegant" \
  -F "season=summer" \
  -F "scene_hint=beach" \
  -F "enable_quality_check=false" \
  -F "use_workflow=true"
```

### 查询任务状态

```bash
curl "http://localhost:8000/api/tasks/{task_id}"
```

## 配置说明

### 模型层级配置

系统支持三种模型层级，可通过 `OVERRUN_MODEL_TIER` 环境变量切换：

| 层级 | 风格分析模型 | 质量评估模型 | 适用场景 |
|------|-------------|-------------|----------|
| `high_quality` | qwen3-vl-8b-instruct | qwen3-vl-8b-instruct | 最终生成，追求最高质量 |
| `standard` (默认) | qwen3-vl-8b-instruct | qwen3-vl-8b-instruct | 日常使用，平衡质量和成本 |
| `economy` | qwen3-vl-8b-instruct | qwen3-vl-8b-instruct | 大批量处理，降低成本 |

### 检索配置常量

| 常量 | 默认值 | 说明 |
|------|--------|------|
| `MIN_SALES_COUNT` | 500 | 最低销量阈值 |
| `MAX_RETRIEVAL_ROUNDS` | 3 | 最大检索轮数 |
| `QUALITY_SCORE_THRESHOLD` | 7.0 | 质量评估阈值 |
| `RETRIEVAL_CANDIDATE_MULTIPLIER` | 4 | 候选数量倍数 |
| `MIN_CANDIDATE_COUNT` | 15 | 最小候选数量 |

### 缓存配置

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `ENABLE_CACHE` | true | 是否启用缓存 |
| `CACHE_MAX_SIZE_MB` | 500 | 缓存最大大小（MB） |

缓存自动清理：每次保存缓存时有 10% 概率触发清理，当缓存超过限制时按 LRU 策略删除旧文件。

## 常见问题

**Q: 检索结果为空？**

A: 循环检索会自动放宽条件，先检查数据库是否有对应品类的商品。

**Q: API Key 验证失败？**

A: 确认 `.env` 文件中 `OPENROUTER_API_KEY` 已正确配置。

**Q: 图像生成超时？**

A: 检查网络连接，FLUX.2 模型可能需要代理访问。设置 `HTTP_PROXY` 环境变量。

**Q: 缓存占用空间过大？**

A: 系统会自动清理超限缓存，也可手动调用 `utils.clear_cache()` 清理。

**Q: Windows 控制台中文乱码？**

A: 系统已内置 `console_utils.fix_console_encoding()` 自动修复。
